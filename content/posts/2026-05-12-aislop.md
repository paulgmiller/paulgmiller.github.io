Talking to some friends there are decent number of people not reviewing code or just having another agent review it.

### Maybe its fine. 

 This is fine for side projects or one off cmds in bigger repos that can be tossed at any time. Yet another argument for microservices. Agents are pretty good at not making and catching simple bugs like nil panics and logic goofs. 
But for code that will continue to grow for a year 

### Did it actually do what you want.  

I've seen agents short cut to make things work. Maybe I'll just return a hard coded string that makes it look like I'm doing work. But they can also take long cuts. Building up whole data storage models when a simple link would work. 

How are you testing? Are you manually going through each change? Do you have e2e? Who reviews the e2e. 

### Did it compromise the code to do so.

 Did it add three if statements when wone would have made more sense. Did it just sling a more code into the file thats already 1000 lines. Are all your members public? 

 When do agents stop to make interfaces. Why not just couple to everything by taking 10 concrete objects as inputs when you type faster than any human.

Many engineers never cared in the first place. But I thought we all agreed those were no hires. 

### Am I wrong. 

We've had agents for what 4 months? Maybe architecture can be put in an agent.md but I'm doubtful. More possible gpt 6 and fable rare just going to be bettter at architecture. But I'm not so sure. So my current predcition is we see alot of projects collapsing in next 6-12 months.
